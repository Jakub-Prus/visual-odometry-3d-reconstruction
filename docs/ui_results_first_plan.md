# UI Results-first Plan

## Why the UI is being refocused
- the app is meant for portfolio demos and public presentation
- visitors should understand the output in under 30 seconds
- final results should appear before technical drill-down

## Primary principles
- result pages lead with trajectory, sparse map, and key metrics
- run launch and progress are first-class product flows
- diagnostics move into deeper pages instead of the main navigation
- demo mode must feel complete even without a backend

## Page priority
1. Home
2. New Run
3. Results
4. Result overview
5. Map, visuals, metrics
6. Technical details

## Primary vs secondary content
- primary: trajectory, point cloud, status, summary metrics, visual artifacts
- secondary: warnings, config JSON, frame tables, fallback logs
