# Docs Override

## Scope
- Applies to `docs/` and its subdirectories, including `architecture/`, `domain/`, `runbooks/`, and `decisions/`.

## Key invariants
- Docs must match current behavior or be clearly marked as planned.
- Prefer links to source files over duplicated technical detail.

## Read first
- Start with the nearest task-related doc and the referenced source file.

## Local rules
- Keep examples short, accurate, and easy to update.
- Put domain knowledge in `docs/domain/*.md`, not in repo-wide instructions.

## Definition of done
- Changed docs are accurate, scoped, and non-duplicative.
